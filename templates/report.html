4 Defensive Recommendations

What should be done next to improve security
  • 1
    Adversarial Training Retrain the models using examples of adversarial inputs. Even one retraining round can cut the evasion rate by 30–60%. This is the single highest-impact fix.
  • 2
    Input Sanitisation (Feature Squeezing) Apply pre-processing to incoming data before it reaches the AI — similar to how a spell-checker smooths out typos. This disrupts the tiny, carefully crafted changes attackers rely on.
  • 3
    Use Multiple Independent Models (Ensemble) If one AI can be fooled, a second structurally different AI probably can't. Running several models in parallel and requiring agreement raises the bar for attackers significantly.
  • 4
    Confidence Thresholding Flag any prediction the AI is not very sure about (below ~75% confidence) and route it to a human reviewer. Attack-crafted inputs often cause the AI to hedge — this catches them.
  • 5
    Anomaly Detection on Inputs Monitor incoming data for unusual statistical patterns. Adversarial examples frequently look subtly "off" compared to normal inputs — an anomaly detector acts as a first-line filter.
  • 6
    API Rate-Limiting & Query Monitoring Black-box attacks (HopSkipJump, ZooAttack) require sending hundreds of queries to probe the AI. Detect and throttle clients making unusually high prediction volumes.
  • 7
    Quarterly Re-Benchmarking Threats evolve. Schedule adversarial benchmarks every quarter with updated attack parameters to ensure defences remain effective over time.

📚 Glossary

Technical terms explained in plain language
Evasion Rate
The percentage of attack attempts that successfully fooled the AI. A rate of 60% means 6 out of every 10 adversarial inputs went undetected. Higher is worse.
Adversarial Attack
A specially crafted input — e.g., a slightly modified malware file or network packet — designed to exploit blind spots in an AI model and cause it to make wrong predictions.
FGSM (White-box)
Fast Gradient Sign Method. Requires full access to the AI's internals. Crafts attacks in a single mathematical step. Represents a worst-case insider or reverse-engineering threat.
HopSkipJump (Black-box)
An attack that needs only the model's predicted label — no internals required. It iteratively walks toward the AI's decision boundary. Realistic for any attacker querying a cloud API.
ZooAttack (Black-box)
A patient, gradient-free black-box attack that probes the model using prediction probabilities. Slow but broadly applicable — represents a persistent, well-resourced external attacker.
Confidence Delta (Δ)
How much the AI's confidence score drops after an attack. A large negative delta means the attack made the AI very uncertain — a sign the input was successfully perturbed.
White-box vs Black-box
White-box: the attacker knows the model's weights and architecture. Black-box: the attacker can only observe inputs and outputs. Black-box is more realistic in the real world.
Adversarial Training
Retraining the AI model using adversarial examples so it learns to recognise and correctly classify them. The most effective known defence against adversarial attacks.
